**How the GPIO 5 conflict was handled**
- The provisional User_Setup_ST7789_240x320 uses TFT_CS = GPIO 5.
- To avoid simultaneous use of GPIO 5 by MAX485 DE/RE, I implemented a build-time override:
  - When CONFIG_DISPLAY_ST7789_240X320 is selected, CMakeLists.txt adds the compiler definition BOARD_RS485_PHY_ENABLED=0 for the main component.
  - mstp_rs485.c already gates actual RS-485 TX and DE handling on BOARD_RS485_PHY_ENABLED; with the macro set to 0 at compile-time the code avoids configuring DE pin, avoids gpio_set_level/de toggles, and MSTP transmit paths early-return.
- When CONFIG_DISPLAY_HW657A is selected, BOARD_RS485_PHY_ENABLED remains unchanged (default 1) and existing RS-485 behavior is preserved.
- This approach is reversible at build time and does not modify MS/TP source logic or state machines.

  mstp_rs485.c

  CMakeLists.txt
